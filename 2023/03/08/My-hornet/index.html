My hornet

我的马蜂十分的奇怪啊，半 google 不 google 的。

头文件

include

includeincludeinclude 顺序按照字典序排列。

includeincludeinclude 后应当空行

不要 使用万能头

尽量少使用头文件

define

正常时候 不使用 definedefinedefine，使用 pairpairpair 的时候 可以 使用 definedefinedefine 来代替 structstructstruct 变量名的作用

namespace std

要 使用 using namespace std;

类型名

要 使用 usingusingusing 语法对类型重命名

long longlong\ longlong long 需要重命名为 LLLLLL

pair<T1, T2> 要被命名为 P(大写的T1类型首字母)(大写的T2类型首字母)

举个例子，using PIL = pair<int, LL>;

要 和 using namespace std; 放在一起，不空行。

常数

类似于 eps,πeps, \pieps,π 这类型的常数，要命名为 kkk + 大写的首字母。

比如，const double kE = 1e-12; (kEkEkE 代表 epsepseps)

要和两个 usingusingusing 的东西放在一起。然后空一行。

代码内容

数组

要 使用 vectorvectorvector 存储，使用 resize()resize()resize() 函数初始化。

函数

不要 提前声明，不要 使用 inlineinlineinline，函数类型能用 boolboolbool 就绝对不用 intintint

sortsortsort 时的自定义函数 要 使用 lambdalambdalambda 且写在 sortsortsort 函数内部。

结构体

能使用一层 pairpairpair 就 绝对不使用 结构体，甚至能用 arrayarrayarray 且不影响美观性也不使用结构体。

使用结构体需要用到运算 要 使用 operatoroperatoroperator 重定义。operatoroperatoroperator 要 写在结构体内部。

大括号不换行

空格

关键字函数和括号之间 要 添加空格

自定义函数 不要 添加空格

运算符两边 要 添加空格

小括号和大括号之间 要 添加空格

命名空间名和双冒号之间 要 添加空格，但双冒号和所属的函数或变量间 不要

双小于号和双大于号两边 要 添加空格

像 #include <iostream> 要添加空格

读写

要 使用 C++C++C++ 读写（cincincin, coutcoutcout），不要 使用快读快写

要 开同步流

小数 要 提前 setprecisionsetprecisionsetprecision 好

遍历方式

如果是非代替数组的 vector，尽量使用 for (auto i : v) 此类容器遍历方式

下标从 111 开始，数组空间卡满，不开大哪怕一点。

版本 : C++ 14 / C++ 17

注释

正文基本不使用，但是也许写题解会写一点？
